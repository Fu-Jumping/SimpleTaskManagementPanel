# 需求与验收说明

本文档用于说明作业2的功能范围、页面范围、数据规则和最终验收标准。开发过程中如出现时间不足或功能不稳定，以本文档中的主链路和降级规则为准。

## 一、题目要求摘要

题目要求实现一个简易任务管理面板，形式类似 Trello。

前端要求：

- 使用 Vue 3 + Vite。
- 使用 provide/inject 或 Pinia 进行组件通信。
- 使用 `<draggable>` 实现拖拽。
- 支持快捷键，例如 Del 删除任务。

功能要求：

- 看板模式包含待处理、进行中、已完成三列。
- 每列可添加、删除、编辑任务卡片。
- 任务卡片包含标题、描述、截止日期、优先级。
- 支持卡片在不同列间拖拽移动。
- 任务数据可导入、导出 JSON。

API 要求：

- 自建 Node + Express + lowdb 或 SQLite。
- 实现任务 RESTful 接口。

加分项：

- 用户登录注册。
- JWT 认证。
- 任务搜索与筛选。
- 数据持久化，刷新后保留。

## 二、项目定位

项目名称可使用：

- 个人任务看板
- 中文任务看板
- 任务管理面板

项目目标：

- 完成一个可演示的个人待办任务管理系统。
- 重点保证完整链路稳定，而不是追求复杂功能。
- 原型只作为视觉和交互参考，正式项目用 Vue 重新实现。

完成档位：

- 完整冲刺。
- 计划用 3-5 天集中完成主要开发和联调。
- 如果时间不足，优先保证演示稳定。

## 三、用户角色与演示故事

主要用户：

- 普通个人用户。

演示故事：

1. 用户打开系统。
2. 用户注册账号。
3. 用户登录进入个人任务看板。
4. 用户查看系统生成的示例任务。
5. 用户新增一个任务。
6. 用户编辑任务标题、描述、截止日期、优先级和状态。
7. 用户在状态看板中拖拽任务，将任务从待处理移动到进行中或已完成。
8. 用户切换到优先级视图，按高、中、低优先级管理任务。
9. 用户搜索或筛选任务。
10. 用户导出当前筛选任务。
11. 用户导入 JSON 任务数据。
12. 用户刷新页面后，确认任务仍然存在。
13. 用户退出登录。

## 四、页面范围

必须实现的页面或视图：

| 页面或视图 | 说明 |
| --- | --- |
| 登录页 | 输入用户名和密码登录 |
| 注册页 | 输入用户名和密码注册 |
| 主看板页 | 展示任务、搜索筛选、状态看板和优先级看板 |
| 任务抽屉 | 用于新增和编辑任务 |
| 数据中心 | 用于任务导入和导出 |
| 用户菜单 | 显示用户名，包含数据中心、生成示例数据、退出登录 |

路由规则：

- 未登录访问主应用时，跳转登录页。
- 已登录访问登录页或注册页时，跳转主看板页。

移动端要求：

- 基础可用。
- 不要求完整移动端体验。
- 不能出现明显横向溢出或按钮无法点击。

## 五、功能需求

### 1. 登录注册

要求：

- 支持用户注册。
- 支持用户登录。
- 使用 JWT 保存登录状态。
- JWT 有效期为 7 天。
- 密码使用 bcrypt 哈希存储。
- 不同用户之间任务数据隔离。

验收标准：

- 可以注册新用户。
- 可以登录已有用户。
- 错误密码不能登录。
- 登录后能进入看板。
- 退出登录后不能继续访问主页面。

### 2. 任务 CRUD

任务字段：

- 标题
- 描述
- 截止日期
- 优先级
- 状态

系统字段：

- `createdAt`
- `updatedAt`
- `order`

要求：

- 支持新增任务。
- 支持编辑任务。
- 支持删除任务。
- 新任务插入对应状态列顶部。
- 删除任务必须弹出确认。
- 按钮删除和 Del 快捷键删除使用同一确认逻辑。

验收标准：

- 新增任务后能立即显示。
- 编辑后页面内容更新。
- 删除确认后任务消失。
- 取消删除后任务保留。
- 刷新后任务仍存在。

### 3. 状态看板

列：

- 待处理
- 进行中
- 已完成

接口值：

- `todo`
- `doing`
- `done`

要求：

- 每列显示对应状态任务。
- 支持同列内拖拽排序。
- 支持跨列拖拽修改任务状态。
- 拖拽后同步后端。

验收标准：

- 任务拖到其他状态列后，状态更新。
- 刷新后拖拽结果仍保留。
- 同列排序结果刷新后仍保留。

### 4. 优先级看板

列：

- 高
- 中
- 低

接口值：

- `high`
- `medium`
- `low`

要求：

- 按优先级分列展示任务。
- 支持拖拽修改任务优先级。
- 优先级视图拖拽只改变 `priority`，不改变 `status`。

验收标准：

- 任务拖到其他优先级列后，优先级更新。
- 任务原有状态不被优先级拖拽改变。
- 刷新后优先级拖拽结果仍保留。

### 5. 搜索筛选

筛选条件：

- 关键词
- 状态
- 优先级

要求：

- 支持多个条件组合筛选。
- 关键词匹配标题和描述。
- 主页面显示筛选结果数量。
- 标题和描述中的关键词需要高亮。

验收标准：

- 输入关键词后只显示匹配任务。
- 同时选择状态和优先级后结果正确收窄。
- 清空筛选后恢复全部任务。

### 6. 导入导出

导入要求：

- 从 JSON 文件导入任务。
- 导入方式为追加，不覆盖已有任务。
- 不做导入前预览。
- 非法项跳过。
- 只显示成功数量和跳过数量。

导出要求：

- 默认导出当前筛选任务。
- 导出格式为：

```json
{
  "exportedAt": "2026-06-04T14:00:00.000Z",
  "count": 1,
  "tasks": []
}
```

验收标准：

- 可以导出 JSON 文件。
- 导出数量与当前筛选结果一致。
- 可以导入合法任务。
- 非法任务不会进入系统。
- 导入后刷新页面仍保留。

### 7. 示例数据

要求：

- 新用户首次登录或注册后生成示例任务。
- 用户菜单提供“生成示例数据”入口。
- 生成示例数据只追加，不覆盖已有任务。

降级：

- 如果该功能不稳定，可以隐藏入口。

## 六、数据规则

字段命名：

- API 和前端统一使用英文驼峰。

校验规则：

| 字段 | 规则 |
| --- | --- |
| `title` | 必填，建议 1-50 个字符 |
| `description` | 可选，建议不超过 500 个字符 |
| `dueDate` | 可选；填写时必须是合法日期 |
| `priority` | 只能是 `high`、`medium`、`low` |
| `status` | 只能是 `todo`、`doing`、`done` |
| `order` | 数字，由后端生成或排序接口更新 |

排序规则：

- 手动拖拽后的 `order` 优先。
- 同一状态列或优先级列内按 `order` 升序展示。

刷新策略：

- 登录后拉取一次任务。
- 新增、编辑、删除、拖拽、导入后刷新任务列表或同步 store。

## 八、最终验收清单

最终演示至少需要通过以下流程：

- 注册新用户。
- 登录已有用户。
- 未登录访问主页面跳转登录页。
- 已登录访问登录或注册页跳转看板页。
- 新增任务。
- 编辑任务。
- 删除任务。
- 点击选中任务后按 Del 弹出删除确认。
- 状态看板拖拽改变状态和排序。
- 优先级看板拖拽改变优先级和排序。
- 优先级拖拽不改变任务状态。
- 关键词搜索。
- 状态筛选。
- 优先级筛选。
- 组合筛选。
- 导入任务。
- 导出当前筛选任务。
- 刷新页面后数据保留。
- 退出登录。

## 十、风险与降级

优先级：

1. 注册、登录、任务 CRUD、状态看板、数据持久化。
2. 优先级看板、搜索筛选、拖拽排序。
3. 导入导出、示例数据、快捷键。

降级规则：

- 影响主链路稳定的功能可以隐藏入口。
- 接口无法按时完成时，优先保证任务 CRUD 和状态拖拽。
- 前端视觉还原不足时，优先保证交互完整。
- 报告以最终实际功能为准。
