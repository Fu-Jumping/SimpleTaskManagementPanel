# 原型参考说明

本文档说明根目录 `design/` 的用途，供前端负责人实现页面时参考。

## 一、目录用途

`design/` 是本项目的视觉原型和参考素材目录，包含前期原型导出的页面代码、样式和截图素材。

当前主要内容：

```text
design/
├── .figma/     # Figma 相关导出内容和截图素材
└── .stitch/    # Stitch 相关导出页面和截图素材
```

说明：

- `design/` 保留在项目根目录，不移动到 `docs/`。
- `.figma/` 和 `.stitch/` 中的导出代码作为视觉参考。
- 正式前端项目仍需在 `client/` 中使用 Vue 3 + Vite 重新实现。
- 不建议直接复制导出代码作为正式代码起点。

## 二、前端参考重点

前端实现时优先参考以下视觉和交互方向：

- 绿色主色调。
- 圆角任务卡片。
- 胶囊按钮和柔和的操作按钮样式。
- 右侧抽屉式的新建和编辑任务表单。
- 状态看板三列布局。
- 优先级看板三列布局。
- 顶部用户菜单和数据中心入口。
- 搜索筛选区域和任务结果数量展示。

## 三、实现要求

正式前端实现以项目文档为准：

- 功能范围以 `docs/requirements.md` 为准。
- 接口字段以 `docs/api.md` 为准。
- 前端任务拆分以 `docs/分工/frontend-tasks.md` 为准。

如果原型效果与需求文档冲突，优先遵守需求文档和接口契约。
